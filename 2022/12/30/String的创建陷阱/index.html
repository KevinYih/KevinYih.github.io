String的创建陷阱

String用new与不new的是有区别的，需理解并掌握原理，并加深对Java内存的理解。

1. 关于String str1 = “abc”

首先在栈(Java Stack)中创建一个String str1，包含变量的类型和引用的变量；接下来，若常量池中有“abc”，则直接指向那个地址；若无，则在字符串常量池中创建一个新的值为“abc”的对象，并将str1指向该地址。

字符串常量池，放在方法区(Method Area)。

1.1. 工作原理

Java内部将此语句转化为以下几个步骤：

(1) 先在栈(Java Stack)中定义一个名为str1的对String类的对象引用变量：String str1。

(2) 在栈(Java Stack)中查找有没有存放值为”abc”的地址，如果没有，则开辟一个存放字面值为”abc”的地址，接着创建一个新的String类的对象o，并将o的字符串值指向这个地址，而且在栈中这个地址旁边记下这个引用的对象o。如果已经有了值为”abc”的地址，则查找对象o，并返回o的地址。

(3) 将str1指向对象o的地址。

值得注意的是，一般String类中字符串值都是直接存值的。String str = “abc” 中，其字符串值却是保存了一个指向存在栈中数据的引用。

1.2. 例一：

  • str1和str2指向了同一空间，“abc”只存储了一次

  • str1==str2：根据jdk的说明，只有在两个引用都指向了同一个对象时才返回真值。相当于两个变量的值相同在内存中只存储了一次，所以两个变量的地址相同。

  • str1.equals(str2)：无论存储位置是否相同，只要两个字符串的值相等时为true。

  • jvm创建了两个引用str1和str2，但只创建了一个对象，而且两个引用都指向了这个对象。

1.3. 例二：

  • 赋值的变化导致了类对象引用的变化，str1指向了另外一个新对象。JVM发现在栈中没有存放该值的地址，便开辟了这个地址，并创建了一个新的对象，其字符串的值指向这个地址。

  • str2仍旧指向原来的对象。

2. 关于String str2 = new String(“abc”)

一定会创建一个对象，可能创建两个对象。

2.1. 工作原理

Java内部将此语句转化为以下几个步骤：

(1) 先在栈(Java Stack)中定义一个名为str2的对String类的对象引用变量：String str2。

(2) 然后，在堆(Java Heap)中开辟一个空间，将此空间的地址给str2（即所谓new了第一个对象）。

(3) 接下来，若方法区(Method Area)的常量池中有“abc”，则将这个”abc”的地址给堆(Java Heap)中开辟的空间；若无，则在常量池中新建一个值为“abc”的对象（创建了第二个对象），将这个常量池中新创建的对象给堆中开辟的那个空间。

2.2. 实例

  • str1和str2在堆空间开辟了不同的空间。
  • 以new的方式创建字符串，即使值相同，地址也会不一样。

3. 结论

String str1 = "abc" 中，str1存的是方法区中字符串常量空间的地址；

String str2 = new String(“abc”) 中，str2存的是堆空间的地址，由堆地址再指向方法区中字符串常量空间的地址。
