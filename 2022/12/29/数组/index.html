1. 数组的定义

数组：相同类型的数据的有序集合。

数组元素：数组中的每个数据。

1.1. 数组的声明和创建

Java的方法是语句的集合，它们在一起执行一个功能。

  • 方法是解决一类问题的步骤的有序组合。
  • 方法包含于类和对象中。
  • 方法在程序中被创建，在其他地方被引用。

1.2. 数组元素的访问

数组的长度：arrayRefVar.length。

2. 内存分析

JAVA在程序运行时，在内存中划分5片空间进行数据的存储。

分别是：寄存器(Register)；本地方法区(Native Method Stacks)；方法区(Method Area)；栈(Java Stack)；堆(Java Heap)。

2.1. 寄存器(Register)

位于处理器内部，最快的存取区。

这是最快的存储区，因为它位于不同于其他存储区的地方——处理器内部。但是寄存器的数量极其有限，所以寄存器由编译器根据需求进行分配。你不能直接控制，也不能在程序中感觉到寄存器存在的任何迹象。

2.2. 本地方法区(Native Method Stacks)

Java官方对于本地方法的定义为methods written in a language other than the Java programming language，就是使用非Java语言实现的方法，但是通常我们指的一般为C或者C++，因此这个栈也有着C栈这一称号。一个不支持本地方法执行的JVM没有必要实现这个数据区域。本地方法栈基本和JVM栈一样，其大小也是可以设置为固定值或者动态增加，因此也会对应抛出StackOverflowError和OutOfMemoryError错误。

2.3. 方法区(Method Area)

可以看做是将类（Class）的元数据，保存在方法区里；方法区中保存着被加载过的每一个类的信息：类型信息、类型的常量池 (即运行时常量池)、字段信息、方法信息、类变量(即static变量）、对类加载器的引用、对Class类的引用、方法表。

运行原理：为了运行某程序，首先会将类名传给了jvm。有了这个名字，jvm找到了这个类文件并读入，它从类文件提取了类型信息并放在了方法区中，通过解析存在方法区中的字节码，jvm激活了main()方法，在执行时，jvm保持了一个指向当前类常量池的指针。

  • 方法区是线程共享的；当有多个线程都用到一个类的时候，而这个类还未被加载，则应该只有一个线程去加载类，让其他线程等待；

  • 方法区的大小不必是固定的，jvm可以根据应用的需要动态调整。jvm也可以允许用户和程序指定方法区的初始大小，最小和最大限制；

  • 方法区同样存在垃圾收集，因为通过用户定义的类加载器可以动态扩展Java程序，这样可能会导致一些类，不再被使用，变为垃圾。这时候需要进行垃圾清理。

  • 根据jvm规范的规定，当方法区无法满足内存分配需求时，将抛出OutOfMemoryError异常。

　

2.4. 栈(Java Stack)

位于RAM中，这是一种快速有效的分配存储方法，仅次于寄存器，对象引用存储在栈中创建程序时候，JAVA编译器必须知道存储在堆栈内所有数据的确切大小和生命周期，因为它必须生成相应的代码，以便上下移动堆栈指针。

2.5. 堆(Java Heap)

位于RAM中，存放所有new出来的java对象。

堆不同于栈的好处是：编译器不需要知道要从堆里分配多少存储区域，也不必知道存储的数据在堆里存活多长时间，用堆进行存储分配比用堆栈进行存储存储需要更多的时间。

  • 栈内存存储的是局部变量而堆内存存储的是实体；

  • 栈内存的更新速度要快于堆内存，因为局部变量的生命周期很短；

  • 栈内存存放的变量生命周期一旦结束就会被释放，而堆内存存放的实体会被垃圾回收机制不定时的回收。

3. 数组的初始化

3.1. 静态初始化

创建和赋值，一并完成。

3.2. 动态初始化

分配了空间，但是所有数组元素的值都是默认值，需后期赋值。

3.3. 数组的默认初始化

数组也是引用类型（除了8大基本数据类型，其他都是引用类型），他的元素相当于类的实例变量。因此数组一经分配空间，其中的每个元素也被按照实例变量同样的方式被隐式初始化。

3.4. 数组的四个基本特点

  • 数组a一旦被创建，长度不可被改变。

    • 下标的合法区间：[0, a.length-1]
    • 数组下标越界异常：ArrayIndexOutOfBoundsException

  • 数组元素类型必须相同，不允许出现混合类型。

  • 数组元素可以是任何数据类型，包括8种基本类型，以及引用类型。

  • 数组也是引用类型（除了8大基本数据类型，其他都是引用类型）。数组也可看成是一个对象，数组元素相当于该对象的成员变量。因为Java中的对象是在堆(Java Heap)中的，因此数组无论保存原始类型还是其他对象类型，数组对象都是在堆(Java Heap)中的。（new出来的对象，都是在堆中。）
