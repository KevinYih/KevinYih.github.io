1. 方法的含义

方法 = 功能 = 函数 = funtion

1.1. Java的方法

Java的方法是语句的集合，它们在一起执行一个功能。

  • 方法是解决一类问题的步骤的有序组合。
  • 方法包含于类和对象中。
  • 方法在程序中被创建，在其他地方被引用。

1.2. 方法设计的原则

方法的本意是功能块，为实现某个功能的语句的集合。

  • 方法的原子性：一个方法只完成一个功能,这样有利于我们后期的扩展。

  • 方法的命名原则：首字母小写 + 驼峰规则。

  • main方法尽可能简洁，干净。把公共模块都提取到main方法外面，通过方法调用的方式来实现功能。

1.3. 方法的定义

Java的方法类似于其他语言的函数，是一段用来完成特定功能的代码片段。

方法包含一个方法头(Method Header)和一个方法体(Method Body)。

  • 修饰符：可选部分，告诉编译器如何调用该方法。例如：public, static, final …静态的东西是在类型加载初始时初始化的，并且与类相关；非静态东西一般放在实例的构造函数中初始化，它是与特定类的实例相关的。

    • 没有使用static修饰的成员为实例对象成员。静态成员的使用：通过类名。

    • 不加static修饰的成员是对象成员，归每个实例对象所有。

    • 加static修饰的成员是类成员，可以由一个类直接调用，为所有对象共有。

  • 返回值类型：方法可能有返回值，returnValueType。没有返回值，则该类型为void。return除了返回结果，还能终止方法，例如：return 0。

  • 方法名：首字母小写 + 驼峰规则。原则：见名知义。

  • 参数列表：可有多个参数，也可不包含任何参数。

    • 形参：用于调用时接收数据。
    • 实参：调用时，实际传给方法的数据。

    Java都是值传递，实参值的拷贝，会创建副本；对形参的修改不会影响到实参。

    引用传递是地址传递，会影响方法外部的值；值传递，不会影响方法外部的值。

  • 方法体：包含具体的语句，实现该方法的具体功能。

    • return：方法结束；
    • break：整个循环结束；
    • continue：单词循环结束。
  • 异常抛出

2. 方法的调用

2.1. 静态与非静态方法

2.1.1. 静态方法(Static Method)

静态方法的生命周期跟相应的类一样长，静态方法和静态变量会随着类的定义而被分配和装载入内存中。一直到线程结束，静态属性和方法才会被销毁。（也就是静态方法属于类方法）

  • 有static关键字修饰的方法，又叫类方法。属于类，不属于对象。

  • 静态方法的生命周期从在类装载的时候就开始，不自动进行销毁，会一直存在于内存中，直到Jvm关闭，所以说静态方法是全局的，贯穿整个进程。

  • static内部只能出现static变量和其他static方法；而且static方法中还不能使用this….等关键字，因为它是属于整个类。

  • 在实例化对象之前就可用类名.方法名() 直接调用。（静态属性，静态方法都是属于类的，可以直接通过类名调用）

2.1.2. 非静态方法(Non-Static Method)

非静态方法的生命周期和类的实例化对象一样长，只有当类实例化了一个对象，非静态方法才会被创建，而当这个对象被销毁时，非静态方法也马上被销毁。（也就是非静态方法属于对象）

  • 无static关键字修饰的方法，又称为实例方法，成员方法。

  • “类”是抽象化的概念，只有实例化成具体对象了，才会分配内存，才能拿来用。实例方法的生命周期，从实例化对象开始，一直到实例化对象被注销回收之后结束。

  • 非静态方法可以访问类中的任何成员，也可调用静态方法；而静态方法只能访问类中的静态成员。

  • 因为静态方法在类实例化前就可以使用，而类中的非静态变量必须在实例化之后才能分配内存。

  • 实例方法只能使用对象.方法名()的方式调用。（成员属性，成员方法是属于对象的，必须通过new关键字创建对象后，再通过对象调用）

2.2. 方法的重载（Overload）

同名，不同参，即可构成重载。

原理：存在多个相同名称的方法时，编译器会根据方法参数的个数，类型，排列顺序等去逐个匹配，以选择对应的方法。如果匹配失败，则编译器报错。

  • 方法名必须相同
  • 参数列表必须不同：参数个数，参数类型，排列顺序不同，均可构成重载。

​ 以下情况不构成重载，需要类型的排列顺序不同。
  • 方法的返回类型可同，也可不同。
  • 方法的返回类型不同，不构成重载。
  • 方法可重载无限多个。

2.3. 命令行传参

希望在程序运行时再传递给它消息，这时候就需要用cmd把参数传给main()函数进行实现。

2.4. 可变参数

从JDK1.5开始，Java支持传递同类型参数的可变参数给一个方法。（参数类型相同，数量不确定）

原理：同类的参数的个数可变。

  • 在方法申明中，在指定参数类型后加一个省略号 ... 。
  • 一个方法中只能指定一个可变参数，并且它是方法的最后一个参数。任何的普通参数必须在它之前申明。

2.6. 递归 recursive

2.6.1. 递归的含义

A方法调用A方法，自己调用自己。

  • 将大型复杂的问题转换为与原问题相似的规模较小的问题。
  • 少量程序去描述原问题解决过程所需要的多次重复计算，大大减少代码量。

2.6.2. 递归的结构

  • 递归头：边界条件。什么时候不再调用自身方法。如果没有头，就将陷入死循环。
  • 递归体：什么时候需要调用自身方法。

递归深度过大，会占用大量的空间内容，时间复杂度高。

小量计算可以用递归，较大规模计算禁止使用递归。
