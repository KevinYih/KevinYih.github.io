1. Introduction

JavaScript is a dynamic programming language commonly used for web development.

  • client-side scripting language.
  • runs directly in the web browser.
  • provides a wide range of functionalities.

JavaScript’s versatility is further enhanced by its extensive ecosystem of libraries and frameworks, such as React, Angular, and Vue.js, which simplify and streamline the development process.

JavaScript is also utilized in server-side development through platforms like Node.js.

2. Basic Syntax

2.1. Variables

All numbers in JavaScript are represented as floating-point values. JavaScript represents numbers using the 64-bit floating-point format defined by the IEEE 754 standard.

variable initialization

Use the var keyword only for declaration or initialization, once for the life of any variable name in a document.

JavaScript is untyped language. This means that a JavaScript variable can hold a value of any data type. Unlike many other languages, you don’t have to tell JavaScript during variable declaration what type of value the variable will hold. The value type of a variable can change during the execution of a program and JavaScript takes care of it automatically.

Naming a variable is the same as Java.

Only numbers, letters, $ and _ can be contained in the name of a variable.

2.3. Js Arrays

Java’s arrays are more structured and offer a static, strongly-typed collection, which is beneficial for performance and type safety.

In contrast, JavaScript’s arrays are more flexible and dynamic, with built-in methods that make them powerful for various manipulations. It’s more like arrayList in java. []

Array Methods

No. Method & Description

20 toString()
Returns a string representing the array and its elements.

Document Object Model.

All the html elements is document..

As same as css, javascript has 3 types of quotation, including internal, external and inline.

For css link, <link rel="stylesheet" href="styles.css" />.

we tend to put it inside the <head> element, so that it gets loaded up first before any of the content of our web site, which tends to be inside the body.

Html code would be executed line by line, so we always put the css link right at the top, just below the title usually.

But, for JS Script, we want to put it right at the end just before the closing body tag.
