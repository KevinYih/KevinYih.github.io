1. Introduction

React = Declarative UI Programming

With React, you define the target UI state(s) - not the steps to get there! Instead, React will figure out & perform the necessary steps.

Define the goal, not the steps.

  • Rendering components on a webpage(UI) based on their current state.
  • Keeping the UI in sync with state, by re-rendering(reacting) when state changes.

REACT reacts to state changes by re-rendering the UI.

4. Components, props and JSX

4.1.Components

React applications are entirely made out of components.

Each component has Data, Logic, Appearance.

We build UIs by building multiple components and combing them.

When a component nested in another component, the former is the child component of the latter.

The hierarchy of components show the relationship of child component and the parent component.

4.2. JSX

4.2.1 Introduction

Declarative syntax to describe what components look like and how they work.

We use JSX to tell React what we want to see on the screen, instead how to do things in imperative way.

  • Components must return a block of JSX.
  • JSX is an extension of JS which allows us to embed JS, CSS, and React components into HTML.
  • Each JSX element is converted to a React.createElement function call.
  • We could use React without JSX, but actually every use JSX.

4.2.2. How to use?

JSX works essentially like HTML, but we can enter ‚ÄúJavaScript expressions‚Äú by using { }.

Statements are not allowed (if/else, for , switch)

4.2.3. Returning Syntax

Only one return is allowed, but we can all conditional return.

Not very recommended.

You can use it, if you want return something entirely different.

Ternary operator recommended more.

4.5. Props

4.5.1. properties in object

It stands for properties in object.

Props is an object.

Props is a kind of communication channel to show how we pass data between components, from parent components to child components, but not up or sideways.

Example:

Like arguments passed to regular JS functions.

Any types of value can be a prop.

Props are read-only, they are immutable! If you need to mutate props, you actually need state.

  • Because mutating props would affect parent, creating side-effect.(not pure).
  • Because components have to be pure function in terms of props and state.

4.5.2. one-way data flow

In React, data can only be passed from parent components to child components by using props.

Angular has two-way data flow.

Children lets you manipulate and transform the JSX you received as the [children prop.

Reuse any components anywhere, and give it any content that we want.

Specially used for content text.

It allows us to make our components truly reusable.

5. State

5.1. Using syntax

We change the state , we change the UI. which is like a memory of component.

So, to update a view, we update state.

When state is updated, React will react to the update.

React components has a built-in state object.

The state object is where you store property values that belong to the component.

When the state object changes, the component re-renders.

  • Update the component‚Äôs view (by re-rendering it), give us a way to change part of UI.
  • Persist local variables between multiple renders and re-renders.

Always treat state as immutable in React.

React is all about immutability. We can only update the state using the tools the react give us.
